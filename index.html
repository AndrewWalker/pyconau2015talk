Before I get started...

Answer the following questions

  • Is your code under version control?
  • Do you write tests?
  • Do your tests run regularly?
  • Do you document your software?

What is "Scientific Software"?

  • Tools often commercial / expensive
  • Difficult integration problems
  • Complex dependencies
  • Different maintenance lifecycles
  • Different expectations for quality
  • Written by scientists, not software engineers

Why worry about Interoperability

  • You can't solve a problem without a library/tool
  • Cost
  • Quality
  • Maintainability

Performance

Necessity

Development time is often more expensive than Computing time

Why wouldn't you do this?

  • It's not glamourous
  • It's time consuming to get right
  • It's hard to maintain
  • Often, something similar already exists
  • You probably need to read / write some C code

Approaches

  • Run an external process
  • Write a Python extension module
  • Use a tool to help you write an extension module
  • Use a Foreign Function Interface library

Subprocess

Interoperability via subprocesses

Subprocess


from subprocess import *

def graphviz(srcf, dstf, fmt='png'):
    cmd = 'dot -T%(fmt)s %(srcf)s -o %(dstf)s' 
    cmd = cmd % locals()
    subprocess.check_call(cmd.split())

When should you use it?

  • If the tool you're interacting with has a CLI
  • If performance isn't an issue

scipy.weave

Take a string of C/C++ code and make it callable from Python

scipy.weave


import scipy.weave
import numpy as np

def det22(A):
    code = '''
    double res;
    res = A2(0,0) * A2(1,1) - A2(1,0) * A2(0, 1);
    out[0] = res;
    '''
    out = [ 0.0 ]
    scipy.weave.inline(code = code,
                       arg_names = ['A', 'out'], 
    return out[0]

x = np.array([[1, 2],[3,5]])
print det22(x)

When should you use it?

  • When you've got fragments of C/C++
  • When you're prototyping

When should you avoid it?

  • When debugging may be time consuming
  • When you you need to deploy

Cython

A DSL for integrating C, C++ and Python

When should you use it?

  • If you're exploring a solution space
  • When you need a smooth transition from prototype to production

When should you avoid it?

  • Cython != python != C/C++
